# 政策申报方案生成器

## 任务目标
- 本 Skill 用于：为政策咨询师快速生成专业级企业项目申报规划方案
- 能力包含：深度信息调研、战略分析、项目动态匹配、Excel规划表生成
- 触发条件：用户需要为客户生成申报方案、制定项目规划或输出申报规划表

## 前置准备
- 依赖说明：scripts脚本所需的依赖包及版本
  ```
  openpyxl==3.1.2
  ```
- 输出文件会自动写入当前项目的 `output/` 文件夹；脚本会在需要时自动创建该目录

## 默认流程：单 Sheet 模板填充

**当前默认输出形态**：优先基于 [references/公司项目规划表_空表.xlsx](references/公司项目规划表_空表.xlsx) 生成单 sheet 规划表。

**当前输出目标**：
- `--fill-company-only`：只填模板顶部企业信息区
- 完整模式：填模板顶部企业信息区 + `①产业政策-资质认定类` + `②产业政策-资助类`

### Skill 协同方式

- `$metainflow-doc-parse`：默认前置，用于解析访谈纪要、电子文档、可下载资料
- `$metainflow-enterprise-query`：拿到公司名后默认使用，补充工商细节
- `$metainflow-web-search` / `$metainflow-web-fetch`：默认不自动触发，仅在分析师明确要求搜索或抓取公开网页时使用

### 支持的前置资料输入类型

- 会议访谈纪要、聊天记录、调研纪要
- PDF / Word / Excel / PPT / CSV / Text / HTML
- 图片、截图、营业执照照片、网页截图中的可读企业信息
- 用户直接提供的企业全称、统一社会信用代码、注册号、结构化字段

### 支持的标准化企业信息字段

最终进入 Excel 生成脚本前，所有上游资料都需要整理为以下字段：

- `company_name`
- `registered_capital`
- `registration_date`
- `employee_count`
- `declared_projects`
- `main_business`
- `existing_qualifications`
- `registered_address`
- `intellectual_property`
- `industry`

### 字段合并与输入优先级

1. 用户明确提供的字段优先使用
2. 文档解析结果次优先使用
3. `$Metainflow Enterprise Query` 的工商查询结果只补空缺字段
4. 若文档字段与工商字段冲突，保留文档值，同时在摘要中记录“文档值 / 工商值 / 待确认”
5. 若只提供公司名，或明确说明“假数据 / 示例 / 先跑通流程”，可进入示例模式
6. 示例模式下，缺失字段用脚本内置 mock 数据补齐
7. 非示例模式下，缺失字段写为 `待补充`

### 单 Sheet 模板模式调用

```bash
python3 scripts/generate-plan.py \
  --fill-company-only \
  --company '{"company_name":"xxx"}' \
  --output 项目申报规划表.xlsx
```

**输出路径规则**

- 输出文件统一规范到当前项目的 `output/` 目录
- 最终文件名统一改为 `{公司名}-{原文件名}`
- 若传入文件名，例如 `项目申报规划表.xlsx`，实际输出为 `当前项目/output/公司名-项目申报规划表.xlsx`
- 若传入相对路径，例如 `demo/项目申报规划表.xlsx`，实际输出为 `当前项目/output/demo/公司名-项目申报规划表.xlsx`

### 单元格写入规则

- `A1:H1`：`{company_name}\n  项目申报规划表`
- 示例模式标题：`{company_name}\n  项目申报规划表（示例数据）`
- `C2:E2`：企业名称
- `C3:E3`：注册资金
- `C4:E4`：注册时间
- `C5:E5`：社保人数
- `C6:H6`：已申报项目
- `C7:H7`：公司主营业务及产品
- `G2:H2`：企业资质荣誉
- `G3:H3`：注册地址
- `G4:H4`：知识产权情况
- `G5:H5`：所属行业

### 首版假数据演示

**示例输入**

```json
{
  "company_name": "源子（深圳）人工智能有限公司"
}
```

**示例输出逻辑**

- 企业名称：源子（深圳）人工智能有限公司
- 注册资金：100万元（示例）
- 注册时间：2024-01-01（示例）
- 社保人数：20人（示例）
- 已申报项目：暂无（示例）
- 公司主营业务及产品：AI智能体；企业知识库；自动化工具（示例）
- 企业资质荣誉：暂无（示例）
- 注册地址：深圳市南山区（示例）
- 知识产权情况：商标3项；软著2项（示例）
- 所属行业：人工智能 / 软件与信息服务业（示例）

## 操作步骤

### Step 0：资料收集与企业识别

**核心方法论**：先把散落在纪要、文档、图片、截图里的企业信息收拢为一份可继续处理的企业画像初稿，而不是一开始就直接做公开网页搜索。

#### 支持的资料来源

1. **会议访谈纪要**：线下拜访记录、电话访谈纪要、微信整理稿
2. **电子文档**：PDF、Word、Excel、PPT、CSV、TXT、HTML
3. **图片/截图**：营业执照图片、企业介绍截图、官网页面截图、聊天截图
4. **结构化输入**：企业全称、统一社会信用代码、注册号、工商 JSON、用户手工字段

#### 默认处理顺序

1. **文档优先解析**
   - 本地文件或 URL 文档，优先使用 `metainflow parse-doc --file ...`
   - `--output json` 适用于后续 agent/脚本继续处理
2. **图片/截图识别**
   - 先识别可见企业名称、统一社会信用代码、注册地址、主营业务等关键信息
   - 本次 SOP 只规定流程，不新增 OCR 自动化脚本
3. **企业名识别**
   - 优先提取企业全称
   - 若只有简称、片段、别名，也先保留到候选列表
4. **形成企业画像初稿**
   - 优先整理：`company_name`、注册地址、注册资本、成立时间、主营业务、资质、知识产权、已申报项目、员工规模

#### 输出：资料 intake 摘要

参考 [assets/templates/company-info-template.txt](assets/templates/company-info-template.txt) 输出：

- 资料来源
- 文档解析摘要
- 企业名识别结果
- 待确认字段

若连可识别的企业名都无法确定，先停止并要求分析师确认目标企业，再继续下一步。

---

### Step 0.5：Metainflow Enterprise Query 工商补全

**核心方法论**：拿到企业名或注册识别信息后，优先通过工商查询补齐基础画像，而不是手工拼凑工商字段。

#### 路由规则

1. **已知统一社会信用代码或注册号**
   - 直接使用 `metainflow enterprise-query --type business --keyword "..."`
2. **已知完整企业全称**
   - 直接使用 `metainflow enterprise-query --type business --keyword "..."`
3. **只有简称、片段、别名**
   - 先使用 `metainflow enterprise-search --keyword "..."`
   - 多候选时，只展示 `name / oper_name / credit_no / start_date`，等待分析师确认
   - 单候选时，再用候选 `credit_no` 或 `name` 查询详情

#### 缓存与刷新

- 默认建议把当前 thread/session 映射到 `--session-id`
- 当用户明确说 `重新查 / 最新 / 刷新 / 别用缓存` 时，加 `--refresh`

#### 字段补全规则

- `Metainflow Enterprise Query` 负责补充工商基础字段，如：注册资本、注册时间、注册地址、经营范围、法定代表人、经营状态
- 查询结果只补空，不自动覆盖已有文档值
- 若与文档字段冲突，在摘要中保留两边值并标记 `待确认`

#### 输出：工商补全摘要

参考 [assets/templates/company-info-template.txt](assets/templates/company-info-template.txt) 输出：

- 工商补全结果
- 冲突字段与待确认项
- 可进入下一步调研的标准化企业画像

---

### Step 1：深度调研（按需联网，3轮迭代）

**核心方法论**：先基于 Step 0 和 Step 0.5 形成的企业画像继续深挖；默认不自动联网搜索公开网页，只有分析师明确要求时才调用网页搜索/抓取。

#### 联网触发规则

- **默认**：不自动调用 `$metainflow-web-search` / `$metainflow-web-fetch`
- **分析师明确要求搜索公开网页时**：
  - 无具体 URL：先用 `metainflow search-summary --query "..."`
  - 有具体 URL：再用 `metainflow web-crawl --url "..."`

#### 第一轮：企业基础画像核验与补充

**搜索目标**：基于 Step 0.5 已补全的工商信息，核验企业定位，并补充仍然缺失的关键字段。

**必搜信息清单**：
1. **工商信息核验**：企业名称、注册资金、注册时间、注册地址、经营范围、法定代表人、经营状态
2. **经营补充信息**：社保人数、实缴资本、是否有经营异常、是否有公开融资/产品介绍
3. **行业定位**：国民经济行业分类、是否属于战略性新兴产业、是否可归类为人工智能/软件/数字经济方向

**调研关键词模板**（仅在分析师明确要求联网时使用）：
``` 
"{企业名称} 工商信息"
"{企业名称} 注册资金 注册地址"
"{企业名称} 经营范围 主营业务"
```

**第一轮思考评估**：
- 这家企业的行业定位是什么？
- 注册地址属于哪个区？是否有区域政策优势？
- 经营范围中哪些业务可能享受政策支持？

#### 第二轮：政策适配信息搜索

**搜索目标**：基于第一轮结果，推导政策适配方向

**推导搜索方向**：
1. **知识产权深挖**
   - 若第一轮发现有专利/软著 → 搜索专利详情、技术领域、专利价值
   - 若未发现知识产权 → 标注待补充，并搜索行业知识产权申报建议
   ```
   "{企业名称} 专利 软著 商标"
   "{企业名称} 知识产权 发明专利"
   ```

2. **资质荣誉追踪**
   - 搜索已获资质（高新技术企业、科技型中小企业、专精特新等）
   - 搜索已申报项目、已获补贴
   ```
   "{企业名称} 高新技术企业 科技型中小企业"
   "{企业名称} 补贴 资助 项目申报"
   ```

3. **行业政策匹配**
   - 基于第一轮确定的行业 → 搜索该行业的政策支持方向
   ```
   "深圳市 {行业名称} 扶持政策 2025"
   "{行业名称} 补贴 资质认定"
   ```

4. **区域政策挖掘**
   - 基于注册地址所在区 → 搜索区级政策
   ```
   "深圳市{区名} {行业名称} 扶持政策"
   "{区名} 科技创新补贴 租金补贴"
   ```

**第二轮思考评估**：
- 企业现有条件能匹配哪些政策？
- 还需要补充哪些资质或条件？
- 行业和区域有哪些专项政策？

#### 第三轮：针对性深挖

**搜索目标**：针对关键信息缺口进行精准补充

**针对性搜索方向**：
1. **竞品对标分析**
   ```
   "{行业名称} 龙头企业 资质认定"
   "深圳市 {行业名称} 成功申报案例"
   ```

2. **政策时效性确认**
   ```
   "深圳市 {政策名称} 2025 申报时间"
   "深圳市 {政策名称} 最新政策"
   ```

3. **企业深度信息**
   - 若为上市公司或融资企业 → 搜索财报、融资信息
   - 若为科技企业 → 搜索技术团队、研发投入、产品信息
   ```
   "{企业名称} 融资 财报"
   "{企业名称} 产品 技术 团队"
   ```

**第三轮思考评估**：
- 信息是否足够支撑战略分析？
- 是否还有关键缺口需要标注待补充？
- 搜索结果是否需要进一步验证？

#### 输出：企业信息摘要

参考 [assets/templates/company-info-template.txt](assets/templates/company-info-template.txt) 格式输出，每条信息附来源链接。

**待补充字段标注规则**：
- 营收数据（非公开信息）
- 研发投入占比（需企业内部数据）
- 员工学历结构（需企业内部数据）
- 具体产品技术细节（需企业配合提供）

**分析师确认环节**：输出摘要后，等待分析师确认/修正/补充，特别是非公开信息。

---

### Step 2：企业战略分析

**核心方法论**：从企业画像推导政策方向，输出明确的优先级建议。

详细指导见 [references/strategy-analysis-guide.md](references/strategy-analysis-guide.md)

**分析框架**：

#### 一、核心业务定位
- 基于经营范围和产品信息，提炼3条核心业务
- 每条业务需与政策申报方向关联

#### 二、产业机遇与风险应对
- **产业机遇**：结合行业趋势、区域政策、企业优势，明确政策红利窗口
- **风险应对**：针对企业短板，提出资质完善和时间规划建议

#### 三、政策申报优先级规划
- **短期目标（0-6个月）**：可立即启动的项目
- **中期目标（6-12个月）**：需一定准备周期的项目
- **长期目标（1年以上）**：需长期积累的项目

**输出格式**：参考 [assets/templates/strategy-template.txt](assets/templates/strategy-template.txt)

**分析师确认环节**：输出战略分析后，等待分析师确认/调整优先级。

---

### Step 3：项目动态搜索与匹配

**核心方法论**：基于企业特征和政策关键词，实时搜索匹配项目，而非依赖静态项目库。

#### 项目搜索策略

**基础搜索模板**：
```
"深圳市 {企业核心关键词} 扶持政策 2025"
"深圳市 {行业} 补贴 项目申报"
"{区名} {行业} 专项政策"
```

**资质认定类搜索**：
```
"深圳市 高新技术企业认定 条件 流程"
"深圳市 科技型中小企业入库"
"深圳市 专精特新 认定条件"
"深圳市 软件企业评估"
```

**资助类搜索**：
```
"深圳市 研发投入补贴"
"深圳市 创新创业 扶持资金"
"深圳市 人才补贴"
"{区名} 租金补贴"
"深圳市 {行业} 专项补贴"
```

**AI/数字经济专项搜索**（适用于科技企业）：
```
"深圳市 人工智能 扶持政策 模型券"
"深圳市 训力券 算力补贴"
"深圳市 数字经济 专项资金"
"深圳市 软件产业 扶持政策"
```

#### 项目匹配逻辑

1. **排除已有项目**：检查企业已获资质和已申报项目
2. **条件匹配**：基于企业条件（行业、规模、区域、知识产权、营收等）筛选
3. **优先级排序**：基于战略分析的优先级建议排序
4. **时间线规划**：标注申报时间和准备周期

#### 项目信息整理规范

参考 [references/project-database.md](references/project-database.md) 的格式规范，整理每个匹配项目：

| 字段 | 说明 |
|------|------|
| 项目名称 | 官方全称 |
| 申报部委 | 发布/受理机构 |
| 申报时间 | 预估申报窗口 |
| 项目性质 | 认定类/资助类/事前资助/事后资助 |
| 资助金额 | 金额描述 |
| 关键申报条件 | 政策原文摘录 |
| 匹配理由 | 基于企业条件说明为何匹配 |
| 政策来源链接 | 原文出处 |

**输出分类**：
- ①资质认定类
- ②资助类

**分析师审核环节**：输出项目列表后，等待分析师确认/删除/补充。

### Step 4：生成规划表

**默认调用方式**：优先基于空表模板输出单 sheet 文件。

```bash
python3 scripts/generate-plan.py \
  --fill-company-only \
  --company '{"company_name":"xxx","registered_capital":"xxx",...}' \
  --output 项目申报规划表.xlsx
```

**完整模式调用方式**：
```bash
python3 scripts/generate-plan.py \
  --company '{"company_name":"xxx","industry":"xxx",...}' \
  --projects '[{"category":"qualification","project_name":"xxx",...},...]' \
  --output 项目申报规划表.xlsx
```

**说明**：

- `generate-plan.py` 的 CLI 接口保持不变，仍只接收整理后的 `company` 和可选 `projects`
- Step 0 / Step 0.5 的文档解析、工商补全、冲突处理都属于 Excel 生成前准备，不新增脚本参数
- Step 2 的战略分析仍用于分析师判断优先级，但不再作为 Excel 生成脚本参数输入

**输出Excel结构**：
- 单 sheet 模板结果
- 顶部企业信息区 + `①产业政策-资质认定类` + `②产业政策-资助类`
- 输出目录：当前项目的 `output/`

## 注意事项

### 搜索质量保障
- 每轮搜索后必须进行"思考评估"，判断信息是否足够
- 关键信息需附来源链接，确保可溯源
- 矛盾信息需交叉验证，选择权威来源
- 文档解析与工商查询得到的冲突字段，必须显式标记 `待确认`
- `$metainflow-web-search` / `$metainflow-web-fetch` 不是默认动作，仅在分析师明确要求时使用

### 分析专业性保障
- 战略分析需紧密结合企业实际情况，避免泛泛而谈
- 项目匹配需引用政策原文，确保合规性
- 优先级建议需考虑企业发展阶段和资源约束

### 输出规范性保障
- Excel生成需符合标准模板结构
- 项目信息需完整填写所有必填字段
- 待补充字段需明确标注，引导分析师跟进

## 使用示例

**示例：为AI企业生成政策申报方案**
```
用户输入：为源子（深圳）人工智能有限公司做一份高新科技企业的项目规划，需求是融资/补贴、高新科技企业规划

执行流程：
1. Step 0：资料收集与企业识别
   - 解析访谈纪要、文档、截图，提取企业名和基础字段

2. Step 0.5：工商补全
   - 使用 Metainflow Enterprise Query 补齐注册资金、注册地址、经营范围等工商信息

3. Step 1：深度调研
   - 基于已补全企业画像，按需继续核验知识产权、政策方向、区域政策
   
4. Step 2：战略分析
   - 输出短期/中期/长期政策申报规划

5. Step 3：项目匹配
   - 搜索匹配科技型中小企业、高新技术企业、模型券、训力券等项目

6. Step 4：生成Excel规划表
```
