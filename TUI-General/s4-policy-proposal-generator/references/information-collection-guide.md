## 概览

深度调研是政策申报方案生成的基石。本指南提供系统化的信息收集方法论，确保分析师获取全面、准确、可溯源的企业信息。

## 核心理念

**不要一次性搜索所有信息**，而是像专业分析师一样：
1. 先建立企业基础画像
2. 基于已有信息推导新的搜索方向
3. 针对关键缺口进行精准补充
4. 每轮搜索后进行思考评估

**目标**：让搜索结果能够支撑战略分析和项目匹配，而非简单罗列信息。

## 三轮迭代搜索框架

### 第一轮：企业基础画像搜索

**目标**：回答"这家公司是什么"

**搜索重点**：
- 工商基础信息（不可变信息）
- 经营状态（动态信息）
- 行业定位（政策匹配基础）

**输出**：企业基础画像，包含企业名称、注册信息、行业分类、经营状态等。

**判断标准**：
- ✓ 企业行业定位明确
- ✓ 注册地址所在区明确
- ✓ 经营范围清晰
- → 进入第二轮

### 第二轮：政策适配信息搜索

**目标**：回答"这家公司可能匹配什么政策"

**搜索重点**：
- 知识产权（专利、软著、商标）
- 已获资质（高新技术企业、科技型中小企业等）
- 行业政策（基于第一轮行业定位）
- 区域政策（基于第一轮注册地址）

**输出**：政策适配信息清单，包含知识产权状况、资质荣誉、可匹配政策方向。

**判断标准**：
- ✓ 知识产权状况明确（有/无，数量）
- ✓ 已获资质清晰
- ✓ 行业政策方向明确
- ✓ 区域政策优势清晰
- → 进入第三轮

### 第三轮：针对性深挖

**目标**：回答"还缺什么关键信息"

**搜索重点**：
- 竞品对标（同类企业资质情况）
- 政策时效性（申报时间、最新政策）
- 企业深度信息（融资、产品、技术团队）

**输出**：深度信息补充，包含竞品对标、政策时效、企业深度画像。

**判断标准**：
- ✓ 关键信息无重大缺口
- ✓ 政策时效性确认
- ✓ 待补充字段明确标注
- → 输出企业信息摘要

## 必搜信息清单

### A级：必须搜索（不可缺失）

| 信息类别 | 具体字段 | 搜索渠道 |
|----------|----------|----------|
| 工商基础 | 企业名称、注册资金、注册时间、注册地址 | 企查查、天眼查 |
| 经营范围 | 主营业务、经营许可 | 企查查、官网 |
| 行业分类 | 国民经济行业分类、战略性新兴产业分类 | 企查查、行业报告 |
| 知识产权 | 专利、软著、商标数量及详情 | 企查查、国家知识产权局 |
| 资质荣誉 | 高新技术企业、科技型中小企业、专精特新等 | 企查查、高企认定网 |

### B级：应该搜索（影响分析质量）

| 信息类别 | 具体字段 | 搜索渠道 |
|----------|----------|----------|
| 经营状态 | 社保人数、实缴资本、经营异常 | 企查查 |
| 已申报项目 | 历史申报记录、已获补贴 | 企查查、政府采购网 |
| 融资情况 | 融资轮次、投资方 | 企查查、IT桔子 |
| 产品信息 | 核心产品、技术方向 | 官网、招聘网站 |

### C级：可选搜索（提升方案深度）

| 信息类别 | 具体字段 | 搜索渠道 |
|----------|----------|----------|
| 团队信息 | 核心团队、员工规模 | 招聘网站、官网 |
| 竞品对标 | 同行业企业资质情况 | 企查查、行业报告 |
| 区域政策 | 注册地所在区政策 | 区政府官网 |

### D级：待补充字段（非公开信息）

| 信息类别 | 具体字段 | 获取方式 |
|----------|----------|----------|
| 财务数据 | 营收、利润、研发投入占比 | 需企业提供 |
| 人员结构 | 员工学历、技术人员占比 | 需企业提供 |
| 技术细节 | 产品技术方案、核心算法 | 需企业提供 |

## 思考评估框架

### 第一轮搜索后思考

**问题清单**：
1. 这家企业的核心业务是什么？
2. 属于哪个行业？是否属于战略性新兴产业？
3. 注册地址在哪个区？该区有哪些政策优势？
4. 经营范围中哪些关键词可能与政策相关？

**决策树**：
```
是否明确行业定位？
├── 是 → 进入第二轮，搜索行业政策
└── 否 → 补充搜索行业分类、主营业务
    └── 仍不明确 → 标注待确认，询问分析师
```

### 第二轮搜索后思考

**问题清单**：
1. 企业有多少知识产权？是否满足高新技术企业基本条件（需I类知识产权1件或II类知识产权5件）？
2. 企业已获哪些资质？还缺哪些资质？
3. 行业内有哪些专项政策？企业是否符合条件？
4. 注册地区域有哪些专项政策？

**决策树**：
```
知识产权是否足够？
├── 充足 → 重点关注高新技术企业、专精特新
├── 不足 → 推荐先申请软著/专利，制定知识产权规划
└── 无 → 强烈建议知识产权布局，标注为短期目标

是否属于重点扶持行业？
├── 是（AI、集成电路、生物医药等）→ 重点关注专项政策
└── 否 → 重点关注通用政策（高新技术企业、研发补贴等）
```

### 第三轮搜索后思考

**问题清单**：
1. 还有哪些关键信息缺失？
2. 政策时效性如何？哪些项目即将截止？
3. 竞品企业有哪些成功经验可借鉴？
4. 是否需要验证某些矛盾信息？

**决策树**：
```
信息是否足够支撑战略分析？
├── 足够 → 输出企业信息摘要
├── 部分缺失 → 标注待补充字段，继续搜索
└── 关键信息缺失 → 明确标注，引导分析师主动收集

是否存在矛盾信息？
├── 是 → 交叉验证，选择权威来源
└── 否 → 继续后续步骤
```

## 信息质量标准

### 来源可信度分级

| 级别 | 来源类型 | 示例 |
|------|----------|------|
| A级 | 政府官方网站 | gov.cn、科创委官网、工信部官网 |
| B级 | 官方查询平台 | 企查查、天眼查、国家知识产权局 |
| C级 | 行业权威媒体 | 36氪、行业报告、专业媒体 |
| D级 | 企业官网/自媒体 | 官网、公众号、招聘网站 |

**规则**：关键信息优先使用A/B级来源，C/D级来源需交叉验证。

### 信息完整性检查

**必填字段**（不可为空）：
- 企业名称
- 注册资金
- 注册时间
- 注册地址
- 所属行业
- 知识产权数量（可为0）

**有条件必填**：
- 若企业成立满1年 → 需确认是否为高新技术企业
- 若企业有研发活动 → 需确认知识产权情况
- 若注册在重点区域 → 需确认区域政策

### 待补充字段标注规范

**格式**：
```
⚠️ 待补充：
- {字段名称}：{缺失原因}（需{获取方式}）
```

**示例**：
```
⚠️ 待补充：
- 2024年营业收入：非公开信息（需企业提供）
- 研发投入占比：非公开信息（需企业提供财务数据）
- 具体产品技术细节：官网信息有限（需企业提供产品说明）
```

## 常见问题处理

### Q1：搜索不到企业信息怎么办？

**可能原因**：
1. 企业名称有误（简称/曾用名）
2. 企业刚成立，信息未公开
3. 企业名称过于泛化

**处理方案**：
1. 尝试搜索法定代表人、股东名称
2. 标注"信息有限"，建议分析师提供企业完整名称
3. 基于经营范围推导行业和政策方向

### Q2：发现矛盾信息怎么办？

**处理原则**：
1. 优先采信A/B级来源
2. 采信更新时间更近的信息
3. 在摘要中标注"信息存在差异"，建议分析师核实

### Q3：企业属于多个行业怎么办？

**处理原则**：
1. 以主营业务对应的行业为主
2. 在战略分析中说明跨行业特点
3. 搜索多个行业的政策，扩大匹配范围
