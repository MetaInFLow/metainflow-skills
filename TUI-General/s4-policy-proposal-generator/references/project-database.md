## 概览

本文档提供项目搜索方法论和格式规范。实际使用时，应基于企业特征实时搜索匹配项目，而非依赖静态项目库。

## 定位说明

**本文档是"搜索参考模板"，而非"静态项目库"**

- **搜索方法论**：指导如何高效搜索政策项目
- **格式规范**：定义项目信息的标准格式
- **示例数据**：提供格式参考，实际项目需实时搜索

**为什么采用动态搜索**：
1. 政策更新频繁，静态库易过时
2. 不同企业需要的项目差异大
3. 实时搜索能获取最新申报时间和条件

## 项目搜索方法论

### 搜索策略框架

```
企业画像 → 提取搜索关键词 → 搜索政策项目 → 筛选匹配 → 整理输出
```

### 关键词提取方法

#### 从企业信息提取搜索关键词

**基础维度**：
| 企业特征 | 提取关键词 | 示例 |
|----------|------------|------|
| 行业 | {行业} + 扶持政策 | "人工智能 扶持政策" |
| 区域 | {区域} + 科技政策 | "南山区 科技创新" |
| 规模 | {规模} + 补贴 | "中小企业 补贴" |
| 业务 | {业务关键词} + 政策 | "软件企业 扶持政策" |

**组合搜索**：
```
"深圳市 {行业} {区域} 扶持政策"
"深圳市 {行业} 补贴 申报"
"{区名} {行业} 专项政策"
```

| 平台 | 特点 |
|------|------|

3. **参考**：行业媒体、咨询机构

## 项目格式规范

### 项目信息字段

每个匹配项目需包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| category | string | 是 | 项目分类：qualification/funding |
| project_name | string | 是 | 项目官方全称 |
| department | string | 是 | 发布/受理机构 |
| application_time | string | 是 | 申报时间窗口 |
| project_type | string | 是 | 认定类/资助类/事前资助/事后资助 |
| funding_amount | string | 是 | 资助金额描述 |
| key_conditions | string | 是 | 关键申报条件（原文摘录） |
| match_reason | string | 是 | 匹配理由（基于企业条件） |
| source_link | string | 是 | 政策来源链接 |

## 项目分类标准

### ① 资质认定类

**特征**：
- 获得资质证书或官方认定
- 长期有效（通常3年）
- 无直接资金，但有隐性价值（税收优惠、品牌提升）

**典型项目**：
- 国家高新技术企业认定
- 科技型中小企业
- 专精特新中小企业
- 软件企业评估
- ISO体系认证

**分类标识**：`"category": "qualification"`

### ② 资助类

**特征**：
- 直接获得资金支持
- 有申报时间窗口
- 需满足特定条件

**典型项目**：
- 研发投入补贴
- 创新创业扶持资金
- 租金补贴
- 人才补贴
- 行业专项补贴

**分类标识**：`"category": "funding"`

## 搜索结果整理规范

### 信息来源要求

- **关键信息**（申报条件、金额）：必须来自政府官方网站
- **辅助信息**（申报时间、流程）：可参考政策聚合平台
- **所有信息**：必须附来源链接

### 时效性要求

- 申报时间：确认是否为2025年最新信息
- 政策条款：确认是否为当前有效版本
- 金额标准：确认是否为最新标准

### 匹配理由要求

**好的匹配理由**：
```
"企业成立满1年，拥有3项发明专利和5项软著，研发人员占比15%，符合高新技术企业申报条件"
```

**差的匹配理由**：
```
"企业可以申报"（无具体依据）
```

---

## 注意事项

1. **动态搜索优先**：不要依赖静态数据，每次都应实时搜索
2. **来源验证**：关键信息必须来自官方渠道
3. **时效确认**：特别注意申报时间窗口
4. **匹配精准**：匹配理由需基于企业实际情况
