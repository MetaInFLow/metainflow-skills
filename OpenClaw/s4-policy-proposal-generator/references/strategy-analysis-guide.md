## 概览

战略分析是连接企业画像与项目匹配的关键桥梁。本指南提供系统化的分析方法论，确保输出具有指导性的政策申报规划。

## 核心理念

**战略分析不是简单的信息总结**，而是：
1. 基于企业现状推导政策方向
2. 识别企业优势与短板
3. 制定分阶段的申报路径
4. 输出明确可执行的优先级建议

**目标**：让分析师和客户清晰知道"应该做什么"和"先做什么"。

## 分析框架

### 一、核心业务定位

**目标**：提炼企业的核心价值，与政策申报方向关联

**分析方法**：
1. 从经营范围中提取3条核心业务
2. 每条业务需包含：业务描述 + 政策关联词

**示例**：
```
✓ 好的定位：
- AI驱动的智能表格生成平台 → 关联：人工智能专项政策、软件产业政策
- 企业数字化转型解决方案 → 关联：数字经济政策、现代服务业政策
- 低代码开发工具研发 → 关联：技术创新政策、软件企业认定

✗ 差的定位：
- 做软件的（太泛，无法关联政策）
- 提供技术服务（缺少具体方向）
```

**业务定位与政策方向对照表**：

| 业务类型 | 政策方向 |
|----------|----------|
| 人工智能研发 | AI专项政策、模型券、训力券、AI应用示范 |
| 软件开发 | 软件企业认定、软件产业扶持、高新技术企业 |
| 硬件制造 | 先进制造业政策、技术改造补贴、专精特新 |
| 生物医药 | 生物医药专项、创新药研发补贴、医疗器械注册 |
| 新能源 | 新能源汽车补贴、储能政策、绿色低碳政策 |
| 文化创意 | 文化产业政策、设计补贴、版权扶持 |

### 二、产业机遇与风险应对

**目标**：分析外部环境，明确政策红利窗口和企业短板

#### 产业机遇分析

**分析维度**：
1. **行业趋势**：行业是否处于政策风口？
2. **区域优势**：注册地是否有专项政策？
3. **企业优势**：企业已有条件是否能借势？

**输出要求**：
- 结合具体政策和数据
- 明确政策红利窗口
- 与企业实际情况关联

**示例**：
```
✓ 好的分析：
深圳"20+8"产业集群政策将人工智能列为重点扶持方向，2025-2026年投入45亿元政策资金；企业所在的南山区是深圳AI产业核心区，对AI企业有专项租金补贴（60元/㎡/月）和算力补贴；企业已拥有3项发明专利，具备申请高新技术企业的基础条件。

✗ 差的分析：
行业前景好，政策支持力度大。（太笼统，无具体政策）
```

#### 风险应对分析

**分析维度**：
1. **资质短板**：缺少哪些关键资质？
2. **时间风险**：是否有申报时间窗口即将关闭？
3. **条件差距**：距离申报条件还有多远？

**输出要求**：
- 明确短板是什么
- 提出具体应对措施
- 标注时间紧迫性

**示例**：
```
✓ 好的分析：
作为初创企业（成立仅6个月），暂不具备高新技术企业申报条件（需成立满1年），建议优先完成科技型中小企业入库，建立基础资质；知识产权数量不足（当前仅3项发明专利），建议6个月内申请3-5项软件著作权，为高新技术企业申报做准备；社保人数仅1人，建议完善社保缴纳，避免影响资质认定。

✗ 差的分析：
企业资质不足，需要补充。（缺少具体建议）
```

### 三、政策申报与合规降本规划

**目标**：输出分阶段、有优先级的申报路径

#### 优先级规划方法

**判断标准**：

| 优先级 | 判断标准 | 示例 |
|--------|----------|------|
| **立即启动** | 条件已满足或即将满足 | 科技型中小企业入库（无门槛） |
| **6个月内** | 需要一定准备但周期可控 | 软著申请（2-3个月）、租金补贴申请 |
| **6-12个月** | 需要积累条件或等待时间窗口 | 高新技术企业（需成立满1年） |
| **1年以上** | 需要长期积累 | 专精特新小巨人、重大专项 |

**时间线规划**：
```
短期目标（0-6个月）：
- 立即可申报项目
- 基础资质建设

中期目标（6-12个月）：
- 需要积累条件的项目
- 进阶资质申报

长期目标（1年以上）：
- 需要长期积累的项目
- 重大专项/标杆项目
```

## 优先级规划方法

### 决策树方法

```
企业成立时间是否满1年？
├── 是 → 可申报高新技术企业
│   └── 知识产权是否充足（I类1件或II类5件）？
│       ├── 是 → 高新技术企业（立即启动）
│       └── 否 → 先申请软著/专利，6个月后申报高企
└── 否 → 优先科技型中小企业入库
    └── 成立满1年后再申报高企

是否有研发活动？
├── 是 → 可申报研发补贴、创新券
│   └── 研发投入是否达标？
│       ├── 是 → 研发补贴（立即关注申报时间）
│       └── 否 → 积累研发投入，明年申报
└── 否 → 关注通用补贴（租金补贴、人才补贴）

是否属于重点扶持行业？
├── 是（AI/集成电路/生物医药等）→ 重点关注专项政策
│   └── 是否有算力需求？
│       ├── 是 → 申请训力券、模型券
│       └── 否 → 关注其他专项
└── 否 → 关注通用政策（高新技术企业、研发补贴）

注册地是否在重点区域？
├── 是 → 关注区域专项政策
│   └── 是否有租金支出？
│       ├── 是 → 申请租金补贴
│       └── 否 → 关注其他区域政策
└── 否 → 关注市级通用政策
```

### 优先级矩阵

| 紧急度 \ 重要度 | 高重要度 | 中重要度 | 低重要度 |
|----------------|----------|----------|----------|
| **高紧急** | 立即启动 | 优先安排 | 有余力再做 |
| **中紧急** | 本月启动 | 近期规划 | 长期关注 |
| **低紧急** | 提前准备 | 持续跟踪 | 保持了解 |

**示例**：
- 高紧急+高重要：即将截止的高新企业申报
- 高紧急+中重要：租金补贴申请（季度截止）
- 低紧急+高重要：知识产权布局（需要时间积累）

```
【风险应对】
- 资质短板：{缺少的资质和建议补充方案}
- 时间风险：{即将关闭的申报窗口}
- 条件差距：{距离申报条件的差距和弥补建议}

三、政策申报优先级规划

【短期目标（0-6个月）】
1. {项目1}：{申报理由} | 时间窗口：{时间}

3. {基础建设}：{知识产权/资质建议}

【中期目标（6-12个月）】
1. {项目1}：{申报理由} | 准备工作：{准备事项}
2. {项目2}：{申报理由} | 准备工作：{准备事项}

【长期目标（1年以上）】
1. {项目1}：{申报理由} | 积累方向：{需要积累的内容}

四、合规降本建议
- {认证/体系建设建议1}
- {认证/体系建设建议2}
```

## 常见场景分析

### 场景1：初创企业（成立不满1年）

**特点**：
- 成立时间短
- 知识产权少或无
- 资质空白
- 资金压力大

**分析重点**：
1. 优先完成科技型中小企业入库（无门槛）
2. 制定知识产权规划（软著为主）
3. 关注创业补贴、租金补贴
4. 关注专项补贴（如AI企业的模型券、训力券）
5. 标注高新技术企业申报时间（成立满1年后）

**输出示例**：
```
【短期目标】
1. 科技型中小企业入库：建立基础资质，为后续申报做准备
2. 申请软著3-5项：为高新技术企业申报积累知识产权
3. 申请租金补贴：降低运营成本

【中期目标】
1. 高新技术企业认定：成立满1年后申报，享受15%税率优惠
2. 专项补贴：根据行业申请AI专项补贴

【长期目标】
1. 专精特新中小企业：积累2-3年后申报
```

### 场景2：成长期企业（成立1-3年）

**特点**：
- 有一定经营基础
- 知识产权有一定积累
- 可能已有部分资质

**分析重点**：
1. 立即申报高新技术企业（如条件满足）
2. 关注研发补贴、创新券
3. 关注专精特新中小企业
4. 关注行业专项政策

### 场景3：成熟期企业（成立3年以上）

**特点**：
- 经营稳定
- 知识产权丰富
- 资质较完善

**分析重点**：
1. 专精特新小巨人
2. 重点专项、重大专项
3. 总部认定、产业链龙头
4. 行业标杆项目

### 场景4：AI/科技企业

**特点**：
- 算力需求大
- 模型采购成本高
- 技术人才密集

**分析重点**：
1. 模型券、训力券
2. AI应用示范项目
3. 软件企业认定
4. 高新技术企业
5. 算力补贴、云服务补贴

## 注意事项

### 分析质量标准

**好的战略分析应具备**：
1. **具体性**：引用具体政策名称和数据
2. **关联性**：与企业实际情况紧密关联
3. **可操作性**：建议明确可执行
4. **时效性**：考虑申报时间窗口

**避免的问题**：
1. 泛泛而谈，没有具体政策
2. 脱离企业实际，生搬硬套
3. 优先级不明确，眉毛胡子一把抓
4. 忽略时间窗口，错过申报机会

### 与项目匹配的衔接

战略分析输出后，项目匹配应：
1. 严格按照优先级排序
2. 引用战略分析中的判断依据
3. 补充具体的申报条件原文
4. 明确时间线和准备事项
