## 示例1：首次创建 Account Plan

### 输入

```text
触发：帮同步齿科生成客户计划
补充：无现有 AP，需要直接落到飞书知识库
```

### 预期执行

1. `feishu-bitable` 查询客户档案表，确认客户编号和历史项目
2. `metainflow-enterprise-query` 刷新工商、资质和股东信息
3. `metainflow-web-search` 补充近期融资、产品、行业动态
4. `feishu-bitable` 汇总项目总表、商机表、反馈表、方案管理表、合同表
5. `feishu_search_doc_wiki` + `feishu_drive_file` 检索拜访纪要、方案、反馈记录；确认无现有 AP
6. 分析决策链、政策机会、价值评估、风险和跟进策略
7. `feishu-create-doc` 创建 AP 文档；必要时用 `feishu_drive_file` 调整目录位置，或直接创建到 wiki 节点
8. `feishu-bitable` 回写客户等级、标签、AP 更新时间和机会摘要

### 预期输出

```markdown
文档结果：
- 已创建《Account Plan – 同步齿科》，并挂入客户知识库节点

更新摘要：
- 新建 10 个标准章节
- 新增 3 个政策机会
- 新增 1 条高风险预警

关键发现：
- 决策人：李总；评估人：张主任
- P0 机会：专精特新
- 风险：高敏感反馈未闭环、报价竞争

待补充：
- 部分联系人手机号
- 近期合同信息
```

## 示例2：S28 商机回调触发更新

### 输入

```text
触发：S28 回调
record_id：QF-C-0042
payload：识别到“同步齿科想做两化融合”，并附带来源反馈摘要
```

### 预期执行

1. `feishu-bitable` 通过 `record_id` 定位客户
2. 仅增量查询商机表、客户档案表和已有 AP
3. 识别“两化融合”是否已经存在于商机表
4. 若不存在，则新增机会；若已存在，则更新状态与金额
5. 更新 AP 的政策机会、价值评估、跟进策略和更新记录
6. 在新增内容中标注 `日期 + S28反馈回调`

### 预期输出

```markdown
文档结果：
- 已更新《Account Plan – 同步齿科》

更新摘要：
- 政策机会新增：两化融合
- 潜在机会金额上调
- 跟进策略改为先做预评估

关键发现：
- 两化融合为新增机会，预估价值 ¥6万-¥10万
- 与既有专精特新项目存在协同推进空间

待补充：
- 客户内部负责信息化的联系人
```

## 示例3：S4/S5 完成后自动更新

### 输入

```text
触发：S4/S5 完成后自动回调
document_id：QF-P-0079
record_id：QF-C-0042
说明：专精特新方案 v2 已完成
```

### 预期执行

1. 通过 `record_id` 获取客户主档
2. `feishu-bitable` 增量读取方案管理表，确认 v2 版本状态
3. `feishu_search_doc_wiki` / `feishu_wiki_space_node` / `feishu_drive_file` 定位已有 AP
4. 更新时间线，追加“方案 v2 完成”
5. 结合方案结论，更新政策机会优先级、价值评估和跟进策略
6. 写入更新记录：`日期 + S4/S5方案完成`

### 预期输出

```markdown
文档结果：
- 已更新《Account Plan – 同步齿科》

更新摘要：
- 历史交互时间线新增 1 条方案完成记录
- 跟进策略更新为推进报价阶段

关键发现：
- 当前推进重点从机会识别转为报价和内部协调
- 价值评估更适合按年度机会池口径汇总

待补充：
- 报价反馈时间窗口
- 客户内部审批链最新状态
```
