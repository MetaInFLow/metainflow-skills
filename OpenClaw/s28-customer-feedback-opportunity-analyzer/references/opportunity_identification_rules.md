## 信号类型定义

| 信号类型 | 定义 | 商机价值 |
|---------|------|---------|
| **新资质申报** | 客户表达申请新资质/项目的意向 | 高 - 新项目机会 |
| **增购/加项** | 客户表达追加服务/产品的意向 | 高 - 增购机会 |
| **续约** | 客户表达继续合作的意向 | 中 - 续约机会 |
| **转介绍** | 客户愿意推荐给他人 | 中 - 新线索机会 |
| **不满转机会** | 客户提出条件性继续合作 | 高 - 挽留机会 |

## 触发语义

### 新资质申报

**触发关键词**：
- "想申请"、"打算申请"、"准备申请"
- "专精特新"、"小巨人"、"高新"、"国高"
- "单冠"、"双软"、"研发中心"
- 具体政策名称

**示例表述**：
- "明年想申请专精特新"
- "我们打算做个小巨人认定"
- "有没有高新申报的服务"

### 增购/加项

**触发关键词**：
- "还能不能做"、"能不能追加"
- "多加一个"、"还想做个"
- "追加服务"、"扩展范围"

**示例表述**：
- "能不能追加一个知识产权规划"
- "还想做个品牌项目"
- "后续可能有其他项目要做"

### 续约

**触发关键词**：
- "明年继续"、"下一期"
- "长期合作"、"续签"
- "还会找你们"

**示例表述**：
- "明年继续跟你们合作"
- "下一期还找你们做"
- "希望能长期合作"

### 转介绍

**触发关键词**：
- "推荐给"、"介绍朋友"
- "同行也需要"、"帮我介绍给"
- "我认识一家公司"

**示例表述**：
- "可以推荐给同行朋友"
- "我认识一家公司也有这个需求"
- "帮你们介绍给其他公司"

### 不满转机会

**触发关键词**：
- "如果能解决就继续"
- "满足这个条件就签约"
- "改好就行"

**示例表述**：
- "如果能解决进度问题就继续合作"
- "价格降到合理范围就签约"
- "方案调整好就推进"

## 输出动作

### 商机表新建字段

| 字段 | 填写内容 |
|-----|---------|
| 商机名称 | {信号类型}-{客户名称} |
| 商机阶段 | 初步接触 |
| 商机来源 | 客户反馈 |
| 产品类型 | 根据信号类型推断 |
| 关联客户 | 客户名称 |

### 联动动作矩阵

| 信号类型 | 商机表 | Account Plan | 通知对象 |
|---------|-------|-------------|---------|
| 新资质申报 | ✅ 新建 | ✅ 追加机会 | 方案部 |
| 增购/加项 | ✅ 新建 | ✅ 追加机会 | 销售 |
| 续约 | ✅ 新建 | ✅ 标记续约 | 销售 |
| 转介绍 | ✅ 新建（新线索） | - | 销售 |
| 不满转机会 | ✅ 新建（条件性） | ✅ 追加风险 | 销售主管 |

## 识别流程

```
1. 扫描反馈内容中的机会信号关键词
   ↓
2. 匹配信号类型
   ↓
3. 提取机会信息：
   - 机会描述
   - 预估价值（如有）
   - 建议行动
   - 关联政策/资质
   ↓
4. 生成商机记录
   ↓
5. 联动更新 Account Plan
   ↓
6. 通知相关人员
```

## 示例

### 示例1：新资质申报

**原文**："明年想申请专精特新"

**识别结果**：
| 字段 | 内容 |
|-----|-----|
| 信号类型 | 新资质申报 |
| 机会描述 | 专精特新申报意向 |
| 预估价值 | 待评估 |
| 建议行动 | 3工作日内输出预评估报告 |
| 关联政策 | 专精特新认定 |
| 商机名称 | 专精特新申报-深圳市同步齿科 |

**联动动作**：
- ✅ 商机表新建记录
- ✅ Account Plan 追加"专精特新申报机会"
- ✅ 通知方案部

### 示例2：增购

**原文**："还想追加一个知识产权规划"

**识别结果**：
| 字段 | 内容 |
|-----|-----|
| 信号类型 | 增购/加项 |
| 机会描述 | 知识产权规划增购意向 |
| 预估价值 | 待评估 |
| 建议行动 | 1周内输出知识产权规划方案 |
| 商机名称 | 知识产权规划增购-{客户名称} |

**联动动作**：
- ✅ 商机表新建记录
- ✅ Account Plan 追加"增购机会"
- ✅ 通知销售

### 示例3：转介绍

**原文**："可以推荐给同行朋友"

**识别结果**：
| 字段 | 内容 |
|-----|-----|
| 信号类型 | 转介绍 |
| 机会描述 | 客户转介绍意愿 |
| 建议行动 | 感谢客户，跟进转介绍线索 |
| 商机名称 | 转介绍线索-{客户名称推荐} |

**联动动作**：
- ✅ 商机表新建记录（新线索）
- ✅ 通知销售

### 示例4：不满转机会

**原文**："如果能解决进度问题就继续合作"

**识别结果**：
| 字段 | 内容 |
|-----|-----|
| 信号类型 | 不满转机会 |
| 机会描述 | 条件性续约机会 |
| 条件 | 解决进度问题 |
| 建议行动 | 紧急处理进度问题，挽回客户信任 |
| 敏感度 | 高 |

**联动动作**：
- ✅ 商机表新建记录（条件性）
- ✅ Account Plan 追加"客户流失风险"
- ✅ 通知销售主管

## 注意事项

1. **宁多勿漏**：无法确定是否为机会时，先标记为机会，用户确认卡片中可删除
2. **条件性机会**：不满转机会需要高敏感度标记，优先处理
3. **转介绍线索**：需要新建独立线索，不与原客户关联
4. **预估价值**：如有明确金额信息则填写，否则填写"待评估"
5. **建议行动**：需具体可执行，避免笼统表述
