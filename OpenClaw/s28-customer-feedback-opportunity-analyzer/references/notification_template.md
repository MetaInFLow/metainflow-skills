## 卡片类型

| 卡片类型 | 用途 | 触发条件 |
|---------|-----|---------|
| 标准反馈通知 | 推送反馈处理建议 | 每条需处理的反馈 |
| 高敏感度升级 | 超时未处理升级通知 | 24h/48h 未处理 |
| 商机识别通知 | 通知新商机线索 | 识别到商业机会 |

## 标准反馈通知卡片

### 卡片内容要素

| 要素 | 必填 | 说明 |
|-----|-----|-----|
| 反馈摘要 | ✅ | 一句话概括反馈内容 |
| 建议动作 | ✅ | 具体告知责任人应做什么 |
| 处理时限 | ✅ | 高24h / 中3工作日 / 低下次跟进前 |
| 关联文档链接 | ✅ | Account Plan、方案、报价的直达链接 |
| 操作按钮 | ✅ | "已处理" / "转交" / "需协助" |

### 卡片预览

```
┌─────────────────────────────────────┐
│ 🔴 客户反馈处理提醒                    │
├─────────────────────────────────────┤
│ **客户**：深圳市同步齿科医疗股份有限公司  │
│ **敏感度**：高                         │
│ **反馈摘要**：报价偏高，竞品低30%        │
│                                       │
│ **建议动作**：48h内输出调整方案         │
│ **处理时限**：2026/03/17               │
├─────────────────────────────────────┤
│ [查看详情] [已处理] [转交] [需协助]    │
└─────────────────────────────────────┘
```

### 内容规范

**反馈摘要**：
- 长度：不超过 50 字
- 格式：客户原话概括，保留关键信息
- 示例："报价偏高，竞品低30%"

**建议动作**：
- 必须具体可执行
- 包含明确的时间节点
- 示例："48h内输出调整方案，同步主管审批"

**处理时限**：
- 高敏感：24小时内
- 中敏感：3个工作日内
- 低敏感：下次跟进前

## 高敏感度升级卡片

### 卡片内容要素

| 要素 | 必填 | 说明 |
|-----|-----|-----|
| 原责任人 | ✅ | 未处理该反馈的责任人 |

```
│ **请关注并协助处理**                   │

```

## 卡片元素说明

### 敏感度图标

| 敏感度 | 图标 | 颜色建议 |
|-------|-----|---------|
| 高 | 🔴 | 红色强调 |
| 中 | 🟡 | 黄色提示 |
| 低 | 🟢 | 绿色标记 |

### 操作按钮功能

| 按钮 | 功能 | 回调动作 |
|-----|-----|---------|
| 查看详情 | 打开关联文档 | 无 |
| 已处理 | 标记反馈为已处理 | 更新多维表状态 |
| 转交 | 转交给其他人处理 | 更新责任人 |
| 需协助 | 请求他人协助 | 通知协作者 |

### 按钮回调说明

**注意**：按钮回调需要配置飞书机器人，接收用户点击事件后更新多维表。

回调流程：
1. 用户点击"已处理"按钮
2. 飞书推送事件到机器人回调地址
3. 机器人调用多维表 API 更新记录状态
4. 回复用户确认消息

```json
          "type": "default",

          "tag": "button",
          "text": {
```

### 变量说明

| 变量 | 说明 | 示例值 |
|-----|-----|-------|

| `{{feedback_summary}}` | 反馈摘要 | 报价偏高，竞品低30% |
| `{{suggested_action}}` | 建议动作 | 48h内输出调整方案 |
| `{{deadline}}` | 处理时限 | 2026/03/17 |
| `{{doc_link}}` | 文档链接 | https://feishu.cn/docx/xxx |
| `{{feedback_id}}` | 反馈编号 | FB-00000001 |

## 使用示例

### 卡片生成与发送

1. 先按本模板整理卡片变量：

```json
{
  "customer_name": "深圳市同步齿科医疗股份有限公司",
  "sensitivity": "高",
  "feedback_summary": "报价偏高，竞品低30%",
  "suggested_action": "48h内输出调整方案",
  "deadline": "2026/03/17",
  "doc_link": "https://feishu.cn/docx/xxx",
  "feedback_id": "FB-00000001"
}
```

2. 委派 `feishu-card` 生成 `interactive card JSON`
3. 委派 `feishu-im` 发送消息：
   - `msg_type`: `interactive`
   - `receive_id`: 责任人的真实 `open_id` / `chat_id`
   - `content`: 字符串化后的卡片 JSON

---

## 注意事项

1. **卡片长度**：反馈摘要和建议动作应简洁，避免卡片过长
2. **链接有效性**：确保文档链接可访问
3. **按钮回调**：需要单独配置机器人回调地址或事件处理服务，本 Skill 只负责卡片内容设计
4. **敏感度标注**：高敏感卡片应突出显示，建议使用红色或醒目标题
5. **时区处理**：处理时限应考虑客户和团队的时区差异
